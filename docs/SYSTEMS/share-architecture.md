# Share Architecture

> Event sharing from app to any platform via universal links.
> Owner: `src/lib/shareSSOT.ts`, `src/lib/deepLinks.ts`, `src/lib/config.ts`

---

## Share URL

**Canonical format:** `https://www.openinvite.cloud/event/:id`

Generated by `getEventUniversalLink(eventId)` in `shareSSOT.ts` (line 95). Points to the website event page, which:
1. Serves OG metadata for social previews (server-side `generateMetadata()`)
2. Renders the full event page for human visitors
3. Triggers AASA universal-link interception on iOS when the app is installed

**Old format:** `https://go.openinvite.cloud/share/event/:id` — legacy bridge domain. The backend `sharePages.ts` handles this: human visitors get a 301 redirect to `www.openinvite.cloud/event/:id`, crawlers (Facebook, Twitter, WhatsApp, Slack) get OG-tagged HTML served inline. The website middleware (`middleware.ts`) passes `go.openinvite.cloud` through without redirect. Deep link parsing in the app handles both formats for backward compat.

## Domain Architecture

| Domain | Purpose |
|--------|---------|
| `www.openinvite.cloud` | Website, AASA host, share destination |
| `openinvite.cloud` (apex) | 308 redirect → `www.openinvite.cloud` |
| `go.openinvite.cloud` | Legacy share domain, served by same website + backend crawler redirect |
| `api.openinvite.cloud` | Backend API |

## FORBIDDEN_DOMAINS

Defined in `shareSSOT.ts` (lines 34-38):
```
api.openinvite.cloud
open-invite-api.onrender.com
go.openinvite.cloud
```

Every share-text builder calls `assertNoForbiddenDomains()` before returning. If a forbidden domain leaks into share text, it throws in dev and logs a warning in production. This is a regression guard — share links must always use `www.openinvite.cloud`.

- OG image: event cover photo or default banner

- Smart app banner: `apple-itunes-app: app-id=6757429210, app-argument=open-invite://event/{id}`

## Universal Links (AASA)

**File:** `openinvite-website/app/.well-known/apple-app-site-association/route.ts`

```json
{
  "applinks": {
    "apps": [],
    "details": [{
      "appID": "T3LP6XXS49.com.vibecode.openinvite.0qi5wk",
      "paths": ["/event/*"]
    }]
  }
}
```

Only `/event/*` paths trigger universal link interception. All other website paths open in the browser.

**Android App Links:** `openinvite-website/public/.well-known/assetlinks.json`

## Deep Link Parsing

**File:** `src/lib/deepLinks.ts`, function `parseDeepLink(url)` (line 239)

Handles multiple URL formats:

| Format | Type | Example |
|--------|------|---------|
| `open-invite://event/:id` | Custom scheme | Direct app deep link |
| `*/share/event/:id` | Legacy web | Old go.openinvite.cloud links |
| `*openinvite.cloud/event/:id` | Universal link | Current www.openinvite.cloud links |
| `open-invite://invite/:code` | Referral | Referral deep link |
| `open-invite://user/:id` | Profile | User profile |
| `open-invite://circle/:id` | Circle | Circle deep link |
| `*/verify-email?token=xxx` | Email verify | Email verification |

**Auth-gated navigation:** If user is not authenticated when deep link arrives, the route is stored as `pendingDeepLinkRoute` and replayed after auth completes (managed in `_layout.tsx`).

**RSVP intent capture:** Event deep links call `setPendingRsvpIntent({ eventId, status: 'going' })` so the RSVP intent survives through signup flow.

## App Share Surfaces

**Event detail page** (`event/[id].tsx`):

| Surface | Action | Implementation |
|---------|--------|----------------|
| Copy Link | Copies universal link to clipboard | `Clipboard.setStringAsync(url)` |
| Text (SMS) | Opens Messages with pre-filled body | `Linking.openURL(sms:...)` |
| Share | Native share sheet | `Share.share({ message, url })` |
| Host Action Card | Copy Link / Text / Share buttons | `HostActionCard.tsx` |

**SMS URI format** (platform-aware):
- iOS: `sms:&body={encoded}` (ampersand separator)
- Android: `sms:?body={encoded}` (question mark separator)

**Share payload:** Built by `buildEventSharePayload()` in `shareSSOT.ts` (line 106):
```
{title} {date} at {time}

Join us

{universalLink}
```

**SMS body:** Built by `buildEventSmsBody()` in `shareSSOT.ts` (line 131):
```
{emoji} {title} {date} at {time} — you in?

{universalLink}
```

**Referral tracking:** Universal links can include `?ref=CODE` param. `appendReferralParam()` adds it safely. Deep link handler extracts and stores the referral code for post-auth claim.

## Platform Detection (Website)

`detectPlatform()` in `openinvite-website/src/lib/eventTracking.ts` — returns `"ios"`, `"android"`, or `"other"` based on user agent.

Used by `RsvpSection.tsx` and `AppDownloadCta.tsx` to route download CTAs:
- iOS → `https://apps.apple.com/app/id6757429210`
- Android → `https://play.google.com/store/apps/details?id=com.vibecode.openinvite.x0qi5wk`

## Invariants

- ALL share text MUST go through `shareSSOT.ts` builders. Direct URL construction is forbidden.
- FORBIDDEN_DOMAINS must never appear in user-facing share text. `assertNoForbiddenDomains()` enforces this.
- Share URLs always use `www.openinvite.cloud`, never `go.openinvite.cloud` or `api.openinvite.cloud`.
- AASA is served on both `www.openinvite.cloud` and `openinvite.cloud` (middleware allows `.well-known` through without redirect).
- Deep link parser handles both legacy `/share/event/:id` and current `/event/:id` formats.
- SMS URI separator is platform-dependent: `&` for iOS, `?` for Android.

---

## Gotchas

- The `go.openinvite.cloud` domain is in FORBIDDEN_DOMAINS for share text but is still served by the website middleware (for old links in the wild). New shares must never use it.
- Universal links only work when the app is installed. Otherwise the URL opens in Safari/Chrome and the website renders the event page (which is the intended fallback).
- `Share.share()` on iOS uses the `url` field for rich previews (link cards in iMessage). On Android, only `message` is used — the URL must be in the message text.
- Referral params (`?ref=CODE`) are stripped by the deep link handler and stored separately. They don't affect event page routing.
- The AASA `appID` format is `{TeamID}.{BundleID}` — if the bundle ID changes, AASA must be updated or universal links will break.
