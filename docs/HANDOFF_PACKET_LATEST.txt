SUMMARY
-------
Fixed "Text strings must be rendered within a <Text> component" crash
by wrapping all numeric values with String() in Profile screen.

ROOT CAUSE
----------
React Native requires explicit string coercion for numbers inside <Text>.
Numbers were being rendered directly, causing crash on some RN versions.

EXACT LINES CHANGED
-------------------
File: src/app/profile.tsx

     →    {String(stats?.attendedCount ?? 0)}
