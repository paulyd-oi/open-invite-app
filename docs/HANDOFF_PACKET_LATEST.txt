HANDOFF PACKET — Event Detail CTA Polish V1
Date: 2026-03-21

SUMMARY

Improved visual hierarchy of RSVP (Going) and Save buttons on event detail screen for non-host view. Buttons now use the event's own theme color (pageTheme.backAccent) instead of generic STATUS colors. RSVP is clearly primary (outlined when inactive, solid fill when active), Save is clearly secondary (ghost when inactive, tinted when active). Style-only changes — no logic, handlers, or data flow modified.

EXACT FILES CHANGED

1. src/app/event/[id].tsx
   - Going button (~lines 2883-2920):
     * Inactive: transparent bg + 1.5px theme-colored border + theme-colored text/icon
     * Active (going): solid pageTheme.backAccent fill + white text + "Going ✓" + shadow
     * Shadow uses pageTheme.backAccent instead of ET.accentPrimary
   - Save button (~lines 2912-2942):
     * Inactive: transparent bg, 0.5px subtle border, textTertiary icon, textSecondary text
     * Active (interested): pageTheme.backAccent + "18" tinted bg, backAccent + "40" border, theme-colored icon/text
     * Replaced STATUS.interested.* tokens with pageTheme.backAccent derivatives

2. docs/FINDINGS_LOG.md
   - Added entry documenting the CTA polish change

DESIGN RATIONALE

- pageTheme.backAccent is the event-specific theme color resolved from event.themeId via resolveEventTheme()
- ET.accentPrimary was STATUS.going.fg (a generic green #34C759-like) — not tied to the event theme
- Outlined inactive state for RSVP gives it visual prominence via the colored border while preserving the filled state as a clear "you're in" confirmation
- Save button goes from near-invisible to properly ghost-styled, only gaining color when active
- The hex suffix trick (backAccent + "18") creates 9% opacity tint; + "40" creates 25% opacity border
