Date: 2026-03-21

SUMMARY

Fixed two Live Activity bugs: (A) activities never auto-expired after event end time, persisting indefinitely on Lock Screen and Dynamic Island, and (B) the manual toggle in event detail always defaulted to OFF even though auto-start logic activates the activity on focus. Root causes were missing staleDate in the native Activity.request() call and useState(false) for the toggle.

EXACT FILES CHANGED

1. ios/OpenInvite/LiveActivityBridge.swift
   - Added endTimeEpoch parameter to startActivity()
   - On iOS 16.2+: builds ActivityContent with staleDate set to event end time for system auto-expiration
   - Falls back to basic Activity.request() on iOS 16.1

2. ios/OpenInvite/LiveActivityBridge.m
   - Added endTimeEpoch:(double)endTimeEpoch parameter to ObjC bridge method declaration

3. src/lib/liveActivity.ts
   - Added endTime parameter to startLiveActivity(), computes endTimeEpoch (defaults to startTime + 1h)
   - Added cleanupExpiredActivities() function for foreground cleanup of expired activities
   - Added [LIVE_ACTIVITY] DEV diagnostic logs

4. src/app/event/[id].tsx
   - Imported cleanupExpiredActivities
   - Passed endTime to all 3 startLiveActivity() call sites (auto-start on focus, RSVP going, manual toggle)
   - Added foreground cleanup call in useFocusEffect before checking active state
   - Changed liveActivityActive default from false to Platform.OS === "ios"

ROOT CAUSE

Symptom A (no auto-expiration): Activity.request() was called without staleDate or dismissalPolicy. The ActivityKit system had no signal to expire the activity. No JS-side cleanup existed either, so expired activities were never ended from the foreground.

Symptom B (toggle default OFF): useState(false) for liveActivityActive meant the toggle always showed OFF on initial render, regardless of whether the Live Activity was actually running via auto-start.

FIX DETAILS

Native layer (iOS 16.2+): startActivity() now constructs an ActivityContent with staleDate set to the event end time epoch. The system will mark the activity as stale and auto-expire it when the event ends. On iOS 16.1, falls back to the basic Activity.request() path (no staleDate support).

JS cleanup layer: cleanupExpiredActivities() iterates any active Live Activities and ends those whose associated event time has passed. This is called from useFocusEffect in event/[id].tsx on every foreground focus, before checking whether to auto-start a new activity.

Toggle default: Changed from useState(false) to useState(Platform.OS === "ios") so the toggle reflects the expected active state on iOS devices.

Bridge: endTimeEpoch:(double)endTimeEpoch added to the ObjC bridge declaration to pass the epoch value from JS to Swift.
