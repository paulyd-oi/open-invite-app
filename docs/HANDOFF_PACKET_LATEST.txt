SUMMARY
-------
Fixed Welcome screen animation "bounce then snap" issue by configuring
FadeInUp springs with higher damping to prevent overshoot.

ROOT CAUSE
----------
Default FadeInUp.springify() uses low damping which causes:
1. Spring overshoots final position (bounce)
2. Animation clips/snaps when settling, especially on slower devices
3. Combined with SlideInRight/SlideOutLeft transitions, timing conflicts

SOLUTION
--------
Created smoothFadeIn() helper that applies:
- damping(20): High damping prevents overshoot
- stiffness(90): Moderate stiffness for smooth motion
- No more "bounce then teleport to final position"

FILES CHANGED
-------------
1. src/app/welcome.tsx
   - Added smoothFadeIn() helper function after imports
   - Replaced all 12 FadeInUp.springify() calls with smoothFadeIn()

KEY DIFFS
---------
+// ============ ANIMATION HELPERS ============
+// Configure FadeInUp with high damping to prevent overshoot/snap
+// Default springify() can cause "bounce then snap" on some devices
+const smoothFadeIn = (delayMs = 0) =>
+  FadeInUp.delay(delayMs).springify().damping(20).stiffness(90);

All Animated.View entering= props changed from:
  FadeInUp.delay(X).springify()
To:
  smoothFadeIn(X)

WHY THIS WORKS
--------------

- stiffness(90) maintains responsiveness without being too stiff

- No setState during animation window
