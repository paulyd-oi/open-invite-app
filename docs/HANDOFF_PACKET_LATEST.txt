HANDOFF PACKET — Social Tab Swipe Suppression V1
Date: 2026-03-21

SUMMARY

Fixed Social tab (Open Invites) registering swipe/scroll gestures as card taps, accidentally navigating to event detail. Added touch movement tracking to EventCard's Pressable — finger movement >10px in any direction suppresses onPress.

EXACT ROOT CAUSE

EventCard in social.tsx nests a Pressable (tap → navigate to event detail) inside a GestureDetector with Gesture.Pan() (swipe-to-reveal RSVP actions). The Pan gesture has activeOffsetX([-20, 20]) — it only activates after 20px horizontal movement. For shorter swipes, diagonal movements, or vertical scrolls where the Pan gesture fails (failOffsetY=[-10, 10]), the Pressable's onPress still fires on touch-up because no gesture claimed the touch.

This affected all non-own, non-series event cards on the Social tab where swipe-to-reveal is enabled. Own events and series cards (canSwipe=false) were also affected by vertical scroll movements triggering tap.

RUNTIME PATH

1. Card render in Social tab:
   - social.tsx EventCard component (line 123-479)
   - Cards rendered via EventSection → .map() inside ScrollView
   - Each card: GestureDetector(panGesture) > Animated.View > Pressable(onPress=handlePress)

2. Press/gesture handler chain:
   - Touch down → onTouchStart records start position, resets wasSwiping
   - Touch move → onTouchMove checks delta, sets wasSwiping if >10px
   - Pan gesture → onStart sets wasSwiping (for 20px+ horizontal swipes)
   - Touch up → Pressable.onPress fires → handlePress checks wasSwiping flag
   - If wasSwiping: press suppressed, flag reset
   - If not swiping: haptic + router.push to event detail

3. Gesture disambiguation mechanism:
   - Primary: onTouchStart/onTouchMove on Pressable tracks raw finger movement
   - Secondary: Pan gesture onStart callback sets wasSwiping for swipe-to-reveal
   - Threshold: 10px movement in any direction suppresses tap
   - Flag lifecycle: reset on each touch start, set on movement, checked on press

FIX

Touch movement tracking on the EventCard Pressable. Three mechanisms:

A. onTouchStart: Records finger start position (pageX/pageY), resets wasSwiping flag.

B. onTouchMove: Computes dx/dy from start position. If either exceeds 10px, sets wasSwiping=true. This catches vertical scrolls, horizontal swipes, and diagonal movements.

C. Pan gesture onStart: Sets wasSwiping=true via runOnJS when the Pan gesture activates (20px+ horizontal). Belt-and-suspenders with mechanism B.

handlePress: Returns early if wasSwiping is true, logging the suppression. Otherwise fires haptic + navigation as before.
