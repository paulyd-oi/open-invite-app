HANDOFF PACKET — P0 Social Tab Privacy Leak Fix
Date: 2026-03-21

SUMMARY

Fixed P0 privacy bug where PRIVATE and work events (e.g., "Team Meeting") rendered on the social/center tab calendar. Replaced denylist filtering (only excluded isBusy) with strict ALLOWLIST that permits only OPEN and GROUP/CIRCLE events. Private/busy/work events are now excluded from the dataset entirely — they never reach the render layer.

EXACT ROOT CAUSE

social.tsx allEvents memo used shouldOmit() which only checked e.isBusy. Events with visibility="private" but isBusy=false passed through. Additionally, shouldMaskEvent() in eventVisibility.ts returned false for event owners, so even the masking layer showed the full title/details. The combination meant a user's own private "Team Meeting" rendered with full title, "You" badge, and "Private" badge on the social tab.
