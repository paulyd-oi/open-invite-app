ROOT CAUSE

Event detail page works because it makes a separate API call to /api/events/:id/attendees?includeAll=true which returns full attendee objects with image URLs. It also has a fallback path that extracts avatars from event.joinRequests.

Discover's feed endpoints (/api/events/feed, /api/events) return events with joinRequests as undefined or empty — the backend does not populate this nested relation for list/feed queries (performance optimization). So the previous code built avatars exclusively from joinRequests, got an empty array, and rendered nothing but the +N overflow badge (remaining = attendeeCount - 0).

The host user object (event.user with { id, name, image }) IS always populated by the feed API but was never included in the avatar list.

EVENT DETAIL SOURCE

File: src/app/event/[id].tsx
- Primary: useQuery → /api/events/:id/attendees?includeAll=true → normalizes { imageUrl: a.imageUrl ?? a.image }
- Fallback: event.joinRequests filtered to accepted, mapped to { imageUrl: r.user.image }
- Renders via: EntityAvatar with photoUrl={a.imageUrl}

DISCOVER FIX

Avatar builder now:
1. Extracts accepted attendees from joinRequests (when populated — future-proof)
2. Always includes host (event.user) as a guaranteed avatar source
3. Merges both lists and dedupes by user id (host won't duplicate if also in joinRequests)
4. Slices to max 3 displayed avatars
5. Renders each via EntityAvatar with photoUrl={a.image}, fallbackIcon="person-outline"
6. +N badge shows only when remaining = attendeeCount - displayed.length > 0
7. Avatar size increased to 24px so profile photos are clearly distinct from badges

BACKEND CONTRACT NOTE

The Discover feed API DOES NOT include attendee data in joinRequests — this is a known backend optimization for list queries. The host photo (event.user.image) is the only guaranteed avatar source on the feed payload. For full attendee stacks, the backend would need to include a top-N attendees preview in the feed response. Current fix uses the best available data (host photo + any joinRequests when present).
