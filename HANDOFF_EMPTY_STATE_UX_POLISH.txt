Why Conditional Wrapper > New Component?
- Avoids prop drilling (10+ props to new EmptyState component)
- Keeps empty state logic colocated with calendar render
- Easier to maintain (single file, single context)
- No risk of prop type mismatches

Why Check All Four Arrays?
Arrays checked:
1. myEvents (events user created)
2. goingEvents (events user RSVP'd to)
3. localEvents (offline-created events)
4. eventRequests (pending event requests)

Rationale:
- Birthday/work events excluded (auto-generated, not user content)
- Event requests included (need user attention, not truly "empty")
- Comprehensive check ensures no false positives

Why Two CTAs (Create + Discover)?
- Primary (Create): Most direct action for engaged users
- Secondary (Discover): Exploration option for curious users
- "or" divider: Signals equal importance, not hierarchy
- Pattern: Matches industry standards (Twitter, Instagram, etc.)

Why Not Show Empty State in List View?
- List view is a filtering mode (show events in chronological list)
- Global empty check would require mode detection complexity
- List view has its own month-specific empty state (better UX)
- Calendar grid view is primary entry point (show empty there first)

Why Emoji Over Icon Library?
- Zero dependencies (uses Unicode)
- Universal language (no i18n needed)
- Personality + warmth (brand tone match)
- Fast render (no SVG parsing)
- Mobile-native (system emoji rendering)
